\# Roadmap



La roadmap décrit les grandes étapes d'évolution du Dinkum Accessibility Toolkit.



Chaque jalon correspond à une évolution cohérente du framework ou des fonctionnalités.



\---



\# ✅ Jalon 1 — Bootstrap du framework (Terminé)



Objectif : construire les fondations techniques du projet.



\## Réalisé



\- \[x] Création du projet BepInEx

\- \[x] Configuration du projet .NET

\- \[x] Chargement du plugin dans Dinkum

\- \[x] Mise en place de l'architecture du projet

\- \[x] Création du Kernel

\- \[x] Machine à états du Kernel

\- \[x] LoggingService

\- \[x] GameService

\- \[x] Documentation initiale

\- \[x] Premier commit Git



Résultat :



Le framework DAT est opérationnel.



Le plugin est chargé par BepInEx et dispose désormais d'une architecture stable sur laquelle pourront être développées les futures fonctionnalités.



\---



\# 🚧 Jalon 2 — Infrastructure Harmony



Objectif : permettre au framework d'intercepter proprement le comportement du jeu.



Prévu :



\- \[ ] Intégration de Harmony

\- \[ ] Centralisation des patches

\- \[ ] Gestion des erreurs de patch

\- \[ ] Journalisation des patches actifs



Résultat attendu :



Le framework pourra modifier le comportement de Dinkum de manière centralisée et sécurisée.



\---



\# 📋 Jalons futurs



Les jalons suivants seront définis au fur et à mesure de la maturation du projet.



L'objectif est de conserver une roadmap concise, orientée architecture et grandes fonctionnalités plutôt qu'une liste exhaustive de tâches.

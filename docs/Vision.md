\# Vision



Accessibility should never come at the cost of immersion.



The Dinkum Accessibility Toolkit (DAT) is designed as a modular accessibility framework dedicated to Dinkum.



Its purpose is not to modify gameplay, rebalance mechanics or change the identity of the game.



Its purpose is to make Dinkum more accessible while preserving the experience imagined by its creators.



DAT aims to improve visual perception while respecting the original artistic direction of Dinkum.



The framework should feel like an accessibility layer rather than a gameplay modification.



Whenever possible, the player should forget that DAT exists.



\---



\## Design principles



Every feature must satisfy five principles.



\### 1. Optional



Accessibility is personal.



Every feature must be individually configurable and never imposed on the player.



\### 2. Immersive



The best accessibility feature is the one that naturally blends into the game world.



Whenever possible, feedback should integrate with Dinkum's existing visual language.



\### 3. Lightweight



DAT must have a negligible impact on performance.



Accessibility should never become a source of technical degradation.



\### 4. Modular



Each feature should remain independent from the others.



Enabling or disabling one accessibility feature must not affect the rest of the framework.



\### 5. Respectful



DAT exists to accompany the original game—not to replace it.



The artistic direction, atmosphere and gameplay philosophy of Dinkum should always remain recognizable.



\---



\## Long-term vision



DAT is intended to become a long-lived accessibility framework.



Its architecture, documentation and development practices are designed so that new accessibility features can be added without compromising the stability or readability of the project.
